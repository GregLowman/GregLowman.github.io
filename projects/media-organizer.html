Automation Tool Case Study

Media Organizer

A backend automation tool built for a real drone footage business — transforming a flat, unsorted flash drive into a fully organized, platform-ready directory in a single command.

Overview

This tool was built for an independent drone footage company that shoots content across multiple neighborhoods and publishes to both Instagram and YouTube. After every shoot, a flash drive arrives with hundreds of unsorted photos, videos, and caption spreadsheets all mixed together in flat neighborhood subfolders.

Manually sorting that content — separating photos from videos, checking each clip's rotation to determine platform destination, converting captions — was taking significant time after every job. This script automates the entire pipeline in a single command.

The Problem

The client published to two platforms with fundamentally different format requirements. Instagram Reels requires vertical video. YouTube expects horizontal. Drone footage comes off the card mixed together with no labeling, and the rotation metadata is embedded in the file — not visible from the filename or dimensions alone.

On top of that, captions were maintained in CSV spreadsheets that needed to be split into individual text files — one per post — organized separately for photos and videos. Doing all of this by hand after every shoot was slow and error-prone.

Technical Deep Dive

Rotation Detection

The core technical challenge was that video rotation is stored as metadata — not encoded into the pixel dimensions. A portrait clip shot on a drone may report as 1920×1080 but carry a 270° rotation flag. Reading dimensions alone would misclassify it every time. The tool shells out to exiftool via Python's subprocess module, parses the rotation value from the output, and routes accordingly:

Rotation: 0    → YouTube (landscape)
Rotation: 270  → InstaReels (portrait)
Anything else  → Unknown (manual review)

Pipeline Order

The script runs four functions in sequence. build_folders() reads the neighborhood names from the source drive and constructs the full output directory tree before any files are moved. sort() distributes photos, videos, and CSV captions into their respective folders. sort_videos() then inspects each video's rotation metadata and routes it to the correct platform subfolder. Finally, delete_empty() removes the original source folders after everything has been moved.

Caption Processing

Caption CSVs use a prefix convention — P: for photo captions and V: for video captions. The csv_to_txt() function parses each row, strips the prefix, and writes individual .txt files into the appropriate caption folder. Each file is named after the caption text itself, making it immediately usable in upload workflows without any renaming.

Error Handling & Edge Cases

The build_folders() function wraps every os.mkdir() call in a try/except FileExistsError block, so the script can be re-run safely without crashing if any folders already exist. The Unknown video folder handles any clips with unexpected rotation values rather than silently dropping them.

Cross-Platform Compatibility

The client operated on both Windows and macOS. Path handling and exiftool invocation were written to work correctly on both systems. The sorting_software variable allows the executable path to be set once at the top of the file without touching the rest of the code.

Takeaways

Building for a real client with a real recurring workflow is fundamentally different from building for a course. The requirements came from genuine pain — hours of manual work after every shoot — which made the success criteria concrete and unambiguous: does it save time, and does it handle edge cases without breaking?

The rotation metadata problem was a good lesson in not trusting the naive solution. Checking pixel dimensions would have seemed correct until it wasn't. Reading the actual metadata was the right call, and learning to shell out to specialized tools like exiftool rather than reinventing the wheel is a habit that carries into any systems or backend role.
