Automation Tool Case Study

Media Organizer

A backend automation tool built for a real client — sorting drone footage into platform-ready directories automatically.

Pythonexiftool ffprobeCross-platform
View on GitHub

Overview

This tool was built for an independent drone footage company that was manually sorting hundreds of media files after every shoot — separating photos from videos, identifying which clips were portrait vs. landscape, and organizing captions. It was tedious, error-prone, and time-consuming.

The solution was a Python script that automates the entire pipeline: scan, classify, sort, and rename — in one command. This is a backend-only tool with no GUI; the interface is the filesystem itself.

The Problem

The client published content across two platforms with different format requirements. Instagram Reels requires vertical video (portrait orientation). YouTube prefers horizontal (landscape). Drone footage comes off the card in a mixed, flat directory with no organization. Captions were maintained in a CSV that needed to be converted to per-file text for upload workflows.

Doing this manually meant opening every file, checking rotation, moving it to the right folder, and cross-referencing the caption spreadsheet. For a shoot with 200+ files, that's an hour of clerical work after every job.

Technical Deep Dive

Rotation Detection

Video rotation is stored as metadata, not encoded into the actual pixel dimensions. A video shot in portrait on a drone may report as 1920x1080 but with a 90° rotation flag in its metadata. Reading pixel dimensions alone would misclassify it. The tool uses exiftool to extract the rotation value from the file's metadata and applies the correct orientation logic before routing.

Directory Structure

The output directory is built programmatically before any files are moved:

output/
├── instagram/
│   ├── reels/        ← portrait video
│   └── photos/
├── youtube/
│   └── videos/       ← landscape video
└── captions/         ← converted .txt files

Caption Processing

The client maintained captions in a single CSV file. The tool parses each row and writes a corresponding .txt file into the captions directory, named to match the associated media file — ready to be used directly in upload workflows.

Cross-Platform Compatibility

The client used both Windows and macOS machines depending on the shoot. Path handling, exiftool invocation, and file operations were all written to work correctly on both systems without modification.

Takeaways

Building for a real client with a real workflow constraint is fundamentally different from building for a course rubric. The requirements came from actual pain, which made the problem clear and the success criteria concrete: does it save time, and does it handle edge cases without breaking?

The rotation metadata problem specifically was a good lesson — the naive solution (check dimensions) would have been wrong in ways that aren't obvious until you encounter drone footage in the wild. Reading the actual metadata was the right call, and exiftool made it clean.
